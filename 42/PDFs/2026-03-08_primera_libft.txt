Libft
Tu primera libreria

Resumen: Este proyecto consiste en programar una libreria en C.
Tu libreria tendra un monton de funciones de proposito general en las
que se apoyaran tus programas.
Version: 16.6

Capitulo I - Introduccion
Programar en C puede ser aburrido cuando uno no tiene acceso a las
funciones comunes mas utilizadas. Este proyecto te permitira entender
como funcionan estas funciones, como implementarlas y aprender a
utilizarlas. Crearas una libreria propia, que sera muy util porque la
usaras en los siguientes proyectos de C.

Asegurate de ir enriqueciendo tu libft a lo largo de tu cursus.
Sin embargo, cuando utilices tu libreria, asegurate de que todas las
funciones usadas respetan las permitidas por cada proyecto.

Capitulo II - Instrucciones generales
- Tu proyecto debera estar escrito en C.
- Tu proyecto debe estar escrito siguiendo la Norma. Si tienes archivos
   o funciones adicionales, tambien cuentan en la verificacion de la
   Norma y tendras un 0 si hay algun error de norma en cualquiera de
   ellos.
- Tus funciones no deben terminar de forma inesperada (segfault,
   bus error, double free, etc), excepto en comportamientos indefinidos.
   Si ocurre, tu proyecto sera considerado no funcional y recibira un 0.
- Toda la memoria reservada en heap debe liberarse cuando sea necesario.
   No se permiten memory leaks.
- Si el enunciado lo requiere, debes entregar un Makefile que compile
   tus fuentes al output requerido con flags -Wall -Wextra -Werror,
   usando cc, y no debe hacer relink innecesario.
- El Makefile debe contener al menos las reglas: $(NAME), all, clean,
   fclean y re.
- Para entregar bonus debes incluir una regla bonus en tu Makefile.
   Los bonus deben estar en archivos separados _bonus.c / _bonus.h.
   La parte obligatoria y bonus se evaluan por separado.
- Si un proyecto permite usar libft, debes copiar sus fuentes y Makefile
   en un directorio libft y compilar primero la libreria.
- Se recomienda crear programas de prueba, aunque no se entreguen ni se
   evaluen.
- Entrega tu trabajo en tu repositorio Git asignado. Solo ese trabajo se
   evaluara.

Capitulo III - Parte obligatoria
Nombre del programa: libft.a
Archivos a entregar: Makefile, libft.h, ft_*.c
Makefile: NAME, all, clean, fclean, re
Funciones autorizadas: ver detalle abajo
Se permite usar libft: no (todavia no la tienes)
Descripcion:
Escribe tu propia libreria: un conjunto de funciones que sera una
herramienta util a lo largo del cursus.

III.1. Consideraciones tecnicas
- Declarar variables globales esta prohibido.
- Si separas una funcion compleja en auxiliares, usa static para que
   queden en el archivo apropiado.
- Pon todos tus archivos en la raiz de tu repositorio del proyecto.
- Se prohibe entregar archivos no utilizados.
- Todos los .c deben compilar con -Wall -Wextra -Werror.
- Debes usar ar para generar la libreria (libtool prohibido).
- Tu libft.a debe crearse en la raiz del repositorio.

III.2. Parte 1 - Funciones de libc
Debes reimplementar las siguientes funciones originales con prefijo ft_.
Mismo prototipo y comportamiento que man.

Sin funciones autorizadas adicionales:
- isalpha
- isdigit
- isalnum
- isascii
- isprint
- strlen
- memset
- bzero
- memcpy
- memmove
- strlcpy
- strlcat
- toupper
- tolower
- strchr
- strrchr
- strncmp
- memchr
- memcmp
- strnstr
- atoi

Para estas dos puedes usar malloc:
- calloc
- strdup

III.3. Parte 2 - Funciones adicionales

Nombre de funcion: ft_substr
Prototipo: char *ft_substr(char const *s, unsigned int start, size_t len);
Parametros:
- s: la string desde la que crear la substring.
- start: indice de inicio en s.
- len: longitud maxima de la substring.
Valor devuelto:
- La substring resultante.
- NULL si falla malloc.
Funciones autorizadas: malloc
Descripcion:
- Reserva con malloc y devuelve una substring de s.

Nombre de funcion: ft_strjoin
Prototipo: char *ft_strjoin(char const *s1, char const *s2);
Parametros:
- s1: primera string.
- s2: string a anadir a s1.
Valor devuelto:
- Nueva string concatenada.
- NULL si falla malloc.
Funciones autorizadas: malloc
Descripcion:
- Reserva y devuelve una nueva string formada por s1 + s2.

Nombre de funcion: ft_strtrim
Prototipo: char *ft_strtrim(char const *s1, char const *set);
Parametros:
- s1: string a recortar.
- set: caracteres a eliminar.
Valor devuelto:
- String recortada.
- NULL si falla malloc.
Funciones autorizadas: malloc
Descripcion:
- Elimina caracteres de set al principio y final de s1.

Nombre de funcion: ft_split
Prototipo: char **ft_split(char const *s, char c);
Parametros:
- s: string a separar.
- c: delimitador.
Valor devuelto:
- Array de nuevas strings terminado en NULL.
- NULL si falla malloc.
Funciones autorizadas: malloc, free
Descripcion:
- Separa s por delimitador c y devuelve el array de substrings.

Nombre de funcion: ft_itoa
Prototipo: char *ft_itoa(int n);
Parametros:
- n: entero a convertir.
Valor devuelto:
- String que representa n.
- NULL si falla malloc.
Funciones autorizadas: malloc
Descripcion:
- Genera una string con el valor entero recibido.

Nombre de funcion: ft_strmapi
Prototipo: char *ft_strmapi(char const *s, char (*f)(unsigned int, char));
Parametros:
- s: string a iterar.
- f: funcion a aplicar por caracter.
Valor devuelto:
- Nueva string transformada.
- NULL si falla malloc.
Funciones autorizadas: malloc
Descripcion:
- Aplica f a cada caracter de s usando indice y caracter.

Nombre de funcion: ft_striteri
Prototipo: void ft_striteri(char *s, void (*f)(unsigned int, char*));
Parametros:
- s: string a iterar.
- f: funcion a aplicar por caracter.
Valor devuelto: nada
Funciones autorizadas: ninguna
Descripcion:
- Aplica f a cada caracter de s pasando indice y direccion del caracter.

Nombre de funcion: ft_putchar_fd
Prototipo: void ft_putchar_fd(char c, int fd);
Parametros:
- c: caracter a enviar.
- fd: file descriptor de salida.
Valor devuelto: nada
Funciones autorizadas: write
Descripcion:
- Envia el caracter c al fd.

Nombre de funcion: ft_putstr_fd
Prototipo: void ft_putstr_fd(char *s, int fd);
Parametros:
- s: string a enviar.
- fd: file descriptor de salida.
Valor devuelto: nada
Funciones autorizadas: write
Descripcion:
- Envia la string s al fd.

Nombre de funcion: ft_putendl_fd
Prototipo: void ft_putendl_fd(char *s, int fd);
Parametros:
- s: string a enviar.
- fd: file descriptor de salida.
Valor devuelto: nada
Funciones autorizadas: write
Descripcion:
- Envia s y luego un salto de linea al fd.

Nombre de funcion: ft_putnbr_fd
Prototipo: void ft_putnbr_fd(int n, int fd);
Parametros:
- n: numero a enviar.
- fd: file descriptor de salida.
Valor devuelto: nada
Funciones autorizadas: write
Descripcion:
- Envia el numero n al fd.

Capitulo IV - Parte bonus
Si completas la parte obligatoria, puedes hacer bonus para obtener
puntos adicionales.

Estructura obligatoria para lista enlazada:
typedef struct s_list
{
	void			*content;
	struct s_list	*next;
} t_list;

Significado de miembros:
- content: informacion del nodo (void *).
- next: direccion del siguiente nodo o NULL.

Implementa estas funciones:

Nombre de funcion: ft_lstnew
Prototipo: t_list *ft_lstnew(void *content);
Parametros:
- content: contenido del nuevo nodo.
Valor devuelto:
- Nuevo nodo.
Funciones autorizadas: malloc
Descripcion:
- Crea un nodo, inicializa content y next en NULL.

Nombre de funcion: ft_lstadd_front
Prototipo: void ft_lstadd_front(t_list **lst, t_list *new);
Parametros:
- lst: direccion del puntero al primer nodo.
- new: nodo a anadir al inicio.
Valor devuelto: nada
Funciones autorizadas: ninguna
Descripcion:
- Anade new al principio de lst.

Nombre de funcion: ft_lstsize
Prototipo: int ft_lstsize(t_list *lst);
Parametros:
- lst: inicio de la lista.
Valor devuelto:
- Longitud de la lista.
Funciones autorizadas: ninguna
Descripcion:
- Cuenta el numero de nodos.

Nombre de funcion: ft_lstlast
Prototipo: t_list *ft_lstlast(t_list *lst);
Parametros:
- lst: inicio de la lista.
Valor devuelto:
- Ultimo nodo.
Funciones autorizadas: ninguna
Descripcion:
- Devuelve el ultimo nodo de la lista.

Nombre de funcion: ft_lstadd_back
Prototipo: void ft_lstadd_back(t_list **lst, t_list *new);
Parametros:
- lst: puntero al primer nodo.
- new: nodo a anadir al final.
Valor devuelto: nada
Funciones autorizadas: ninguna
Descripcion:
- Anade new al final de lst.

Nombre de funcion: ft_lstdelone
Prototipo: void ft_lstdelone(t_list *lst, void (*del)(void *));
Parametros:
- lst: nodo a liberar.
- del: funcion para liberar content.
Valor devuelto: nada
Funciones autorizadas: free
Descripcion:
- Libera content con del y luego libera el nodo.
- No debe liberar next.

Nombre de funcion: ft_lstclear
Prototipo: void ft_lstclear(t_list **lst, void (*del)(void *));
Parametros:
- lst: direccion del puntero a un nodo.
- del: funcion para eliminar content.
Valor devuelto: nada
Funciones autorizadas: free
Descripcion:
- Elimina y libera lst y todos los nodos siguientes.
- Al final, *lst debe quedar en NULL.

Nombre de funcion: ft_lstiter
Prototipo: void ft_lstiter(t_list *lst, void (*f)(void *));
Parametros:
- lst: puntero al primer nodo.
- f: funcion aplicada al content de cada nodo.
Valor devuelto: nada
Funciones autorizadas: ninguna
Descripcion:
- Itera la lista y aplica f en cada content.

Nombre de funcion: ft_lstmap
Prototipo: t_list *ft_lstmap(t_list *lst, void *(*f)(void *), void (*del)(void *));
Parametros:
- lst: puntero al primer nodo.
- f: funcion aplicada al content de cada nodo.
- del: funcion para liberar content si hace falta.
Valor devuelto:
- Nueva lista resultante.
- NULL si falla malloc.
Funciones autorizadas: malloc, free
Descripcion:
- Itera lst, aplica f y crea una nueva lista con resultados.
- Si hay error de reserva, usa del para limpiar correctamente.

En Makefile debes anadir una regla bonus que incorpore las funciones
bonus a libft.a.
